# Proposal Workflow

## Purpose

Use this guide when the user is preparing a doctoral proposal, especially the first three chapters.

## Minimal Backbone

Lock these items before expanding the draft:

1. Working title
2. Research problem
3. Research questions
4. Theoretical framework
5. Constructs or variables
6. Method choice

If the user says the logic is weak, the problem is usually here.

## Proposal Logic Chain

Write the proposal around this chain:

1. Explain the practical and academic problem.
2. Show what the literature has already established.
3. Identify the gap.
4. Explain why the chosen framework fits the gap.
5. State the research questions.
6. State hypotheses or propositions when needed.
7. Show how the methodology can answer those questions.

## Chapter Expectations

### Chapter 1 Introduction

Include:

1. Research background
2. Problem statement
3. Research objectives
4. Research questions
5. Significance
6. Structure of the study

### Chapter 2 Literature Review

Include:

1. Theme-based review, not paper-by-paper listing
2. Theoretical framework
3. Research gap
4. Hypotheses or propositions

### Chapter 3 Methodology

Include:

1. Research design
2. Sampling and respondents
3. Measures and constructs
4. Data collection procedure
5. Analysis plan
6. Ethics handling

## Quantitative Proposal Rule

If the user already has model testing, questionnaire data, SEM paths, or scale items, strongly prefer explicit hypotheses. A quantitative proposal without hypotheses often feels structurally incomplete.

## Typical Deliverables

- `outputs/proposal_brief.md`
- `outputs/proposal_draft.md`
- `project_config.yaml`
- `research_plan.md`

## Common Failure Modes

1. Title is about AI, but variables are actually about short-video effects.
2. Research questions ask about mechanism, but methods only describe general attitudes.
3. Literature review discusses broad tourism technology and never narrows to the actual model.
4. Method section names SEM, but constructs and paths are still vague.
