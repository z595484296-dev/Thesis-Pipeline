# Gates

## Purpose

Use gates to decide whether a thesis deliverable is ready to move forward.

## Hard Gates

A hard gate failure means stop and repair before proceeding.

Hard gates:

1. The title, questions, framework, and method do not align.
2. A core claim has no traceable evidence.
3. A quantitative design has no clear constructs or paths.
4. A methodology section cannot actually answer the stated questions.
5. A literature review has no explicit gap statement.
6. The draft depends on fabricated citations or invented findings.

## Soft Gates

A soft gate failure means note it and improve later if needed.

Soft gates:

1. Citation count is still thin.
2. Chapter length is below target.
3. Literature breadth is narrow.
4. Wording is repetitive or over-templated.
5. Formatting is incomplete.

## Proposal Completion Check

Call a proposal minimally usable when all of these are true:

1. The working title is stable.
2. The research questions are explicit.
3. The framework is stated and justified.
4. The literature review identifies a gap.
5. The method can test or answer the questions.
6. Ethics handling is acknowledged when required.

## Revision Priority

Fix in this order:

1. logic
2. evidence
3. structure
4. language
5. formatting
